# 进销存管理系统 - 服务器部署指导文档

## 一、服务器环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | >= 14.0.0 | 推荐使用 LTS 版本 |
| MySQL | >= 5.7 | 或使用 Docker 容器运行 |
| Nginx | >= 1.18 | 反向代理和静态文件服务 |
| PM2 | 全局安装 | Node.js 进程管理器 |
| Git | 最新版 | 代码版本管理 |

## 三、首次部署

### 3.1 安装系统依赖

```bash
# 更新系统包
sudo apt update && sudo apt upgrade -y

# 安装 Node.js (使用 NodeSource)
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 安装 PM2
sudo npm install -g pm2

# 安装 Nginx
sudo apt install -y nginx

# 安装 Git
sudo apt install -y git
```

### 3.2 配置 MySQL

**方式一：Docker 运行（推荐）**

```bash
# 拉取 MySQL 镜像
docker pull mysql:5.7

# 启动 MySQL 容器
docker run -d \
  --name mysql-inventory \
  -p 127.0.0.1:3307:3306 \
  -e MYSQL_ROOT_PASSWORD=your_root_password \
  -e MYSQL_DATABASE=inventory \
  -v mysql-data:/var/lib/mysql \
  mysql:5.7
```

**方式二：直接安装 MySQL**

```bash
sudo apt install -y mysql-server
sudo mysql_secure_installation
```

### 3.3 克隆项目代码

```bash
# 创建项目目录
mkdir -p /home/myproject
cd /home/myproject

# 克隆代码
git clone https://github.com/your-username/inventory-system.git
cd inventory-system
```

### 3.4 部署后端

```bash
cd /home/myproject/inventory-system/backend

# 安装依赖
npm install

# 复制环境配置文件（根据实际情况修改）
cp .env.example .env
vi .env

# 初始化数据库
npm run db:init

# 使用 PM2 启动后端
pm2 start app.js --name inventory-backend

# 保存 PM2 配置
pm2 save
pm2 startup
```

### 3.5 部署前端

```bash
cd /home/myproject/inventory-system/web

# 安装依赖
npm install

# 构建生产版本
npm run build
```

### 3.6 配置 Nginx

```bash
# 复制 Nginx 配置文件
sudo cp /home/myproject/inventory-system/deploy/nginx-inventory.conf /etc/nginx/conf.d/

# 测试配置
sudo nginx -t

# 重载 Nginx
sudo nginx -s reload
```

## 四、更新部署流程

### 4.1 备份上传文件（重要！）

```bash
# 备份上传的图片文件（uploads 目录不在版本控制中）
sudo cp -r /home/myproject/inventory-system/backend/uploads /tmp/uploads-backup-$(date +%Y%m%d)
```

### 4.2 拉取最新代码

```bash
cd /home/myproject/inventory-system

# 拉取最新代码
git pull origin main
```

### 4.3 更新后端

```bash
cd /home/myproject/inventory-system/backend

# 安装新依赖（如果有）
npm install

# 重启后端服务
pm2 restart inventory-backend

# 查看日志确认启动成功
pm2 logs inventory-backend --lines 50
```

### 4.4 更新前端

```bash
cd /home/myproject/inventory-system/web

# 安装新依赖（如果有）
npm install

# 重新构建
npm run build
```

### 4.5 恢复上传文件

```bash

```

```bash
sudo cp /home/myproject/inventory-system/deploy/nginx-inventory.conf /etc/nginx/conf.d/inventory.conf

sudo nginx -t

sudo nginx -s reload
```

## 五、Nginx 配置说明

### 5.1 当前配置文件

配置文件位置：`/etc/nginx/conf.d/inventory.conf`

```nginx
# 进销存管理系统 - Nginx 配置
# 将此文件放到服务器 /etc/nginx/conf.d/ 目录下，然后 nginx -s reload
#
# 部署说明：
# - 后端 API：PM2 运行在本机 3001 端口
# - MySQL：Docker 容器运行，映射到 127.0.0.1:3307

server {
    listen 80;
    server_name inventory.lawapp.top;

    # 前端静态文件
    root /home/myproject/inventory-system/web/dist;
    index index.html;

    # 前端路由 - SPA 支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到 PM2 后端（包含上传接口和图片访问）
    # ^~ 前缀匹配优先级高于正则匹配，确保 /api/uploads/xxx.jpg 不会被静态缓存规则拦截
    location ^~ /api/ {
        proxy_pass http://127.0.0.1:3001/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 30s;
        proxy_read_timeout 30s;
        client_max_body_size 10m;
    }

    # 上传文件反向代理
    location ^~ /uploads/ {
        proxy_pass http://127.0.0.1:3001/uploads/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

### 5.2 配置要点说明

| 配置项 | 说明 |
|--------|------|
| `server_name` | 域名，根据实际情况修改 |
| `root` | 前端构建产物目录 |
| `location ^~ /api/` | API 反向代理，`^~` 确保优先级高于正则匹配 |
| `location ^~ /uploads/` | 上传文件代理，避免图片被静态缓存规则拦截 |
| `client_max_body_size` | 上传文件大小限制，默认 10MB |

### 5.3 修改域名

如果需要更换域名，修改 `server_name` 后重载：

```bash
# 编辑配置
sudo vi /etc/nginx/conf.d/inventory.conf

# 修改 server_name 为新域名
# server_name your-new-domain.com;

# 测试并重载
sudo nginx -t && sudo nginx -s reload
```
