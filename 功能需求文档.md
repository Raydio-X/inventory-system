# 服装进销存系统 - 功能需求文档

## 一、项目概述

本系统为面向个人服装店的移动端进销存管理工具，单店铺使用，无需多角色权限管理。系统围绕服装店日常经营中的**批发开单、进货管理、账目核对、欠款跟踪**四大核心场景，帮助店主轻松实现数字化经营。

- **前端**：Vue 3 + TDesign Vue
- **后端**：Node.js (Express/Koa)
- **适配**：移动端优先，响应式布局

### 2.1 账户管理

| 功能 | 说明 |
|------|------|
| 登录/注册 | 账号密码登录，个人使用无需角色区分 |
| 修改密码 | 修改登录密码 |
| 操作日志 | 记录关键操作（出入库、调价、删除等），可追溯 |

### 2.2 商品管理

#### 2.2.1 商品信息维护

| 功能 | 说明 |
|------|------|
| 商品录入 | 录入商品基本信息：名称、品牌、分类、季节、年份等 |
| SKU管理 | 服装核心属性：颜色、尺码，支持多颜色多尺码矩阵管理 |
| 商品图片 | 支持上传多张商品图片，主图/详情图管理 |
| 商品状态 | 上架/下架状态管理，下架商品不出现在开单列表中 |

#### 2.2.2 分类与品牌

| 功能 | 说明 |
|------|------|
| 分类管理 | 多级分类（如：男装/上衣/T恤），支持自定义分类 |
| 品牌管理 | 品牌信息维护，按品牌筛选商品 |
| 标签管理 | 自定义标签（如：新品、热销、清仓），支持多标签筛选 |

#### 2.2.3 批量操作

| 功能 | 说明 |
|------|------|
| 批量导入 | 支持通过 Excel 模板批量导入商品 |
| 批量改价 | 批量修改商品售价或成本价 |
| 批量上下架 | 批量调整商品上下架状态 |

#### 2.2.4 数据处理规则

- 商品删除采用软删除，已产生交易的商品不可硬删除
- SKU 由「商品 + 颜色 + 尺码」唯一确定，同一商品下颜色尺码组合不可重复
- 商品下架后不影响已有库存数据，仅从开单商品列表中隐藏

### 2.3 批发开单（核心）

本模块为系统最核心的操作入口，支持快速完成一笔批发销售的全流程。

#### 2.3.1 商品选择

| 功能 | 说明 |
|------|------|
| 搜索商品 | 按商品名称、品牌、分类关键词搜索 |
| 分类浏览 | 按分类树逐级浏览商品列表 |
| 颜色尺码选择 | 选中商品后，以矩阵方式展示可选颜色×尺码组合，点击即可添加到订单 |
| 最近开单 | 展示最近开单的商品，方便快速重复开单 |
| 收藏商品 | 常开单商品可收藏，置顶显示 |

**界面交互要点：**
- 颜色尺码矩阵：横轴为尺码（S/M/L/XL...），纵轴为颜色，每个交叉格显示库存数量，点击格子弹出数量输入
- 库存为0的SKU在矩阵中灰显，不可点击
- 已选商品在矩阵中高亮标记，显示已选数量

#### 2.3.2 数量录入

| 功能 | 说明 |
|------|------|
| 单SKU录入 | 在矩阵中点击对应格子弹出数字键盘，输入数量 |
| 批量录入 | 支持"整手录入"（如一款所有尺码各1件） |
| 数量调整 | 已选商品列表中可直接增减数量或手动输入 |
| 库存校验 | 输入数量超过当前库存时弹出提醒，可选择继续或调整 |

**数据处理规则：**
- 数量必须为正整数
- 允许超库存开单（批发场景下可能先开单后补货），但需明确提醒
- 修改数量后实时更新订单金额

#### 2.3.3 价格计算

| 功能 | 说明 |
|------|------|
| 默认售价 | 自动填充商品预设售价 |
| 单品改价 | 可针对单个SKU修改单价，支持按件改价或按折扣改价 |
| 整单折扣 | 对整单应用统一折扣（如：全场8折） |
| 按量阶梯价 | 可设置阶梯价格规则（如：10件以上9折，50件以上8折），开单时自动匹配 |
| 手工抹零 | 支持整单抹零操作，记录抹零金额 |
| 价格记忆 | 对同一客户开单时，自动记忆上次该商品的价格，方便快速确认 |

**数据处理规则：**
- 订单金额 = Σ（各SKU数量 × 单价）× 整单折扣 - 抹零金额
- 改价操作记录原价与改后价格，便于后续核对
- 阶梯价优先级：单品改价 > 阶梯价 > 默认售价

#### 2.3.4 客户关联

| 功能 | 说明 |
|------|------|
| 选择客户 | 开单时关联已有客户（按手机号/姓名搜索） |
| 新建客户 | 开单过程中快速新建客户（仅填姓名和手机号即可） |
| 散客开单 | 不关联客户，标记为散客订单 |
| 客户价格 | 关联客户后，自动应用该客户的历史价格或专属折扣 |

#### 2.3.5 付款与订单生成

| 功能 | 说明 |
|------|------|
| 付款方式 | 支持现金、转账、微信、支付宝等多种方式，可组合付款 |
| 部分付款 | 支持先付部分金额，剩余记为欠款（关联未付款核对模块） |
| 全额付款 | 一次性付清，订单状态标记为已结清 |
| 挂单暂存 | 未完成订单可暂存，后续继续编辑结算 |
| 订单确认 | 确认后生成正式销售单，库存实时扣减，账目实时记录 |

**界面交互要点：**
- 开单页面底部固定显示：已选商品数、合计金额、结算按钮
- 结算弹窗展示：商品汇总、金额明细、付款方式选择、实收输入、找零计算
- 部分付款时，欠款金额醒目显示，需二次确认

**数据处理规则：**
- 订单生成后，各SKU库存实时扣减
- 部分付款订单的欠款金额自动记入客户欠款（关联 2.7 未付款核对）
- 订单生成后不可修改，如需调整走退货流程
- 每笔订单生成唯一单号，格式：XS + 年月日 + 4位序号（如 XS202606210001）

#### 2.3.6 销售退货

| 功能 | 说明 |
|------|------|
| 查找原单 | 按单号或客户查找原销售单 |
| 选择退货商品 | 勾选需退货的SKU及数量，支持部分退货 |
| 退货退款 | 退款金额可调整（如扣除折损），支持原路退回或记入客户余额 |
| 库存回补 | 退货确认后，对应SKU库存自动回补 |

**数据处理规则：**
- 退货数量不可超过原单该SKU的购买数量
- 退货后更新原销售单的退款状态和金额
- 退款金额记入账目流水的支出项

### 2.4 进货管理

#### 2.4.1 采购下单

| 功能 | 说明 |
|------|------|
| 创建采购单 | 选择供应商，添加商品SKU及数量、单价 |
| 快速补货 | 基于销售数据或库存预警，一键生成建议采购清单 |
| 采购单状态 | 草稿 → 已下单 → 部分到货 → 已完成 → 已关闭 |

**界面交互要点：**
- 添加商品时支持从商品库选择或直接输入商品名称快速创建
- 采购单列表按状态分Tab展示（全部/待到货/部分到货/已完成）

#### 2.4.2 采购入库

| 功能 | 说明 |
|------|------|
| 确认入库 | 采购到货后，逐项确认实际入库数量（可能与订单数量不一致） |
| 部分入库 | 支持分批入库，每次入库记录实际到货数量 |
| 入库价格确认 | 入库时确认实际采购单价，更新SKU成本价 |
| 自动更新库存 | 入库确认后，对应SKU库存自动增加 |

**数据处理规则：**
- 成本价更新采用移动加权平均法：新成本价 = (原库存金额 + 本次入库金额) / (原库存数量 + 本次入库数量)
- 实际入库数量可能与采购数量不同，以实际入库为准
- 每次入库生成一条库存流水记录

#### 2.4.3 采购退货

| 功能 | 说明 |
|------|------|
| 退货给供应商 | 查找原采购单，选择退货商品及数量 |
| 退款处理 | 记录供应商退款金额及退款方式 |
| 库存扣减 | 退货确认后，对应SKU库存自动扣减 |

#### 2.4.4 供应商管理

| 功能 | 说明 |
|------|------|
| 供应商档案 | 维护供应商信息：名称、联系人、电话、地址、主营品类、备注 |
| 供应商对账 | 查看与某供应商的采购总额、已付金额、未付金额 |
| 采购历史 | 按供应商查看历史采购记录 |

#### 2.4.5 进货数量账目

| 功能 | 说明 |
|------|------|
| 采购汇总 | 按时间段汇总采购总数量、总金额 |
| 按商品汇总 | 查看某商品在各时间段的采购数量及金额 |
| 按供应商汇总 | 查看各供应商的供货数量及金额占比 |
| 进出货对比 | 同一商品采购数量 vs 销售数量对比，辅助判断补货需求 |

**数据处理规则：**
- 采购金额以实际入库确认的金额为准，非下单金额
- 退货金额从采购金额中扣减
- 进出货对比公式：净采购量 = 采购入库量 - 采购退货量；净销售量 = 销售出库量 - 销售退货量

### 2.5 库存管理

#### 2.5.1 实时库存查询

| 功能 | 说明 |
|------|------|
| 按商品查询 | 查看某商品下所有SKU的库存数量及金额 |
| 按分类查询 | 按商品分类汇总库存数量及金额 |
| 按颜色/尺码筛选 | 按颜色或尺码维度筛选库存 |
| 库存金额统计 | 展示库存总数量、总成本金额、总售价金额 |

#### 2.5.2 库存预警

| 功能 | 说明 |
|------|------|
| 预警设置 | 为商品设置库存上限和下限 |
| 低库存提醒 | 库存低于下限时，首页及商品列表中显示预警标识 |
| 高库存提醒 | 库存超过上限时提醒，避免积压 |
| 预警列表 | 专门的预警列表页，集中展示所有预警商品 |

#### 2.5.3 库存盘点

| 功能 | 说明 |
|------|------|
| 创建盘点 | 选择盘点范围（全量/按分类/按品牌） |
| 逐项盘点 | 录入实际盘点数量，系统自动对比账面数量 |
| 差异确认 | 确认盘盈/盘亏，系统自动调整库存 |
| 盘点报告 | 生成盘点差异报告，记录调整明细 |

**数据处理规则：**
- 盘点调整生成库存流水，类型标记为"盘点调整"
- 盘盈：实际 > 账面，库存增加，成本价按当前移动加权平均价计算
- 盘亏：实际 < 账面，库存减少，按当前成本价计算损失金额

#### 2.5.4 库存流水

| 功能 | 说明 |
|------|------|
| 流水查询 | 按时间段、SKU、变动类型查询库存变动记录 |
| 变动类型 | 采购入库、销售出库、采购退货、销售退货、盘点调整 |
| 关联单据 | 每条流水关联对应的采购单/销售单/盘点单号 |

#### 2.5.5 库存关联规则

```
销售开单确认 → 各SKU库存扣减 → 库存不足时提醒（不强制禁止）
采购入库确认 → 各SKU库存增加 → 自动更新移动加权平均成本价
销售退货确认 → 各SKU库存回补 → 退款金额记入账目支出
采购退货确认 → 各SKU库存扣减 → 退款金额记入账目收入
库存盘点确认 → 差异自动调整 → 生成盘盈/盘亏记录及库存流水
```

### 2.7 未付款核对（核心）

本模块专门管理批发业务中的欠款场景，是服装批发店的核心需求。

#### 2.7.1 欠款客户管理

| 功能 | 说明 |
|------|------|
| 欠款客户列表 | 展示所有有未结清欠款的客户，按欠款金额降序排列 |
| 欠款总额 | 顶部醒目展示当前总欠款金额、欠款客户数 |
| 客户欠款详情 | 点击客户查看其所有未结清订单及欠款明细 |
| 欠款趋势 | 展示近30天欠款总额变化趋势，判断资金风险 |

**界面交互要点：**
- 欠款列表每行显示：客户姓名、欠款笔数、欠款总额、最近欠款日期
- 欠款超过7天的记录用红色标记，超过30天的用深红色标记
- 支持按欠款金额区间筛选（如：欠款>5000元的客户）

#### 2.7.2 未结订单跟踪

| 功能 | 说明 |
|------|------|
| 未结清订单列表 | 展示所有未全额付款的销售订单，按开单时间排序 |
| 订单付款状态 | 每笔订单显示：订单金额、已付金额、欠款金额 |
| 部分收款 | 对未结清订单进行部分收款，记录收款金额及方式 |
| 一次性结清 | 对某笔订单一次性付清全部欠款 |
| 按客户汇总 | 查看某客户名下所有未结清订单及总欠款 |

**数据处理规则：**
- 每笔收款生成一条收款记录，关联原销售单号
- 收款后更新销售单的已付金额和欠款金额
- 欠款金额 = 订单金额 - 已付金额
- 全额付清后订单状态更新为"已结清"

#### 2.7.3 收款操作流程

```
进入欠款客户列表 → 选择客户 → 查看未结清订单
    → 选择订单 → 输入收款金额 → 选择收款方式
    → 确认收款 → 更新订单已付/欠款金额
    → 若全额付清 → 订单状态更新为已结清
```

### 2.8 账目核对（核心）

本模块为店主提供日常收支的全面核对能力，确保账目清晰。

#### 2.8.1 收支明细

| 功能 | 说明 |
|------|------|
| 收入记录 | 自动记录所有销售收款、客户还款、其他收入 |
| 支出记录 | 自动记录采购付款、退货退款、其他支出 |
| 手动记账 | 支持手动录入非业务相关的收支（如房租、水电、运费等） |
| 收支分类 | 收入分类：销售收入、客户还款、其他收入；支出分类：采购支出、退货退款、日常费用、其他支出 |
| 收支流水 | 按时间倒序展示所有收支记录，支持按分类、时间段筛选 |

**数据处理规则：**
- 销售开单时，收款金额自动记入"销售收入"
- 部分付款时，实收金额记入"销售收入"，欠款金额不计入收入
- 客户还款时，还款金额记入"客户还款"收入
- 采购入库付款时，付款金额自动记入"采购支出"
- 销售退货退款时，退款金额记入"退货退款"支出
- 手动记账需选择收支分类，备注必填

#### 2.8.2 交易记录查询

| 功能 | 说明 |
|------|------|
| 按时间查询 | 按日/周/月/自定义时间段查询交易记录 |
| 按类型查询 | 按收入/支出/全部类型筛选 |
| 按客户/供应商查询 | 查看与某客户或某供应商相关的所有交易 |
| 按单号查询 | 根据销售单号或采购单号查询关联交易 |
| 关键词搜索 | 按备注关键词搜索交易记录 |

#### 2.8.3 账目统计

| 功能 | 说明 |
|------|------|
| 日汇总 | 每日收入总额、支出总额、净利润 |
| 月汇总 | 每月收入总额、支出总额、净利润，与上月环比 |
| 收支趋势 | 近30天/近12个月收支趋势折线图 |
| 分类占比 | 各收支分类的金额占比饼图 |
| 利润分析 | 销售额 - 成本 - 费用 = 利润，支持按时间段查看 |

#### 2.8.4 对账功能

| 功能 | 说明 |
|------|------|
| 日对账 | 核对当日系统记录与实际资金是否一致 |
| 月对账 | 核对当月系统记录与实际资金是否一致 |
| 对账差异 | 标记差异项，支持手动调整或补充记录 |
| 对账报告 | 生成对账结果，展示期初余额、本期收入、本期支出、期末余额 |

**数据处理规则：**
- 对账公式：期初余额 + 本期收入 - 本期支出 = 期末余额
- 期末余额应等于所有资金账户的当前总额
- 差异项需记录原因说明

#### 2.8.5 对账操作流程

```
选择对账时间段 → 系统自动汇总该时段收支
    → 展示收入明细、支出明细、净利润
    → 店主核对实际资金 → 标记一致/有差异
    → 有差异项 → 补充记录或调整 → 生成对账报告
```

## 三、核心业务流程

### 3.1 批发开单全流程

```
进入开单页 → 搜索/浏览商品 → 选择颜色尺码 → 输入数量
    → 系统自动计算金额 → 可修改单价/应用折扣
    → 选择客户（或散客）→ 确认订单
    → 选择付款方式 → 输入实收金额
    ├─ 全额付款 → 订单已结清 → 库存扣减 → 账目记录
    └─ 部分付款 → 订单部分欠款 → 库存扣减 → 账目记录 → 欠款记入未付款核对
```

### 3.2 采购入库流程

```
创建采购单 → 选择供应商 → 添加商品(SKU)及数量、单价
    → 确认下单 → 到货验收 → 逐项确认实际入库数量
    → 确认入库 → 库存自动增加 → 成本价自动更新 → 账目记录采购支出
```

### 3.3 欠款回收流程

```
进入未付款核对 → 查看欠款客户列表 → 选择客户
    → 查看未结清订单 → 选择订单 → 输入收款金额
    → 选择收款方式 → 确认收款
    → 更新订单已付/欠款金额 → 账目记录客户还款
    → 若全额付清 → 订单状态更新为已结清
```

### 3.4 日终对账流程

```
进入账目核对 → 选择今日 → 系统自动汇总今日收支
    → 展示：销售收款、客户还款、采购支出、退货退款等
    → 核对实际资金 → 确认一致/标记差异
    → 生成日终对账报告
```

### 3.5 库存盘点流程

```
创建盘点任务 → 选择盘点范围 → 逐项录入实际盘点数量
    → 系统对比账面库存 → 生成差异报告
    → 确认调整 → 库存自动修正 → 记录盘盈/盘亏 → 生成库存流水
```

### 3.6 退货流程

```
销售退货：查找原销售单 → 选择退货商品及数量 → 确认退款金额
    → 库存自动回补 → 退款记入账目支出 → 更新客户记录

采购退货：查找原采购单 → 选择退货商品及数量 → 确认退款金额
    → 库存自动扣减 → 退款记入账目收入 → 更新供应商记录
```
